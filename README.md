# Modern Authentication UI Project

This is a clean, modern, and responsive authentication system built using vanilla HTML, CSS, and JavaScript. The project demonstrates a seamless user flow from landing to login, signup, and password recovery.

## 🚀 About the Project

The goal of this project was to create a professional-grade frontend for common user authentication tasks. It features a cohesive design language with smooth transitions and robust client-side validation to enhance the user experience.

### Key Features:
- **Responsive Design**: Fully optimized for mobile, tablet, and desktop screens.
- **Modern Aesthetics**: Uses a vibrant gradient background, Inter font, and subtle animations.
- **Form Validation**: Real-time feedback for required fields, email formatting, and password length.
- **Seamless Navigation**: Interconnected pages with proper relative linking.
- **Centered Layout**: All components are perfectly centered for a clean look.

## 🛠️ Technologies Used

- **HTML5**: For semantic structure.
- **CSS3**: For custom styling, flexbox layouts, and animations.
- **JavaScript (ES6+)**: For form validation and navigation logic.
- **Google Fonts**: Inter font for high readability.

## 📂 File Structure

- `index.html`: The welcome landing page with clear calls to action.
- `login.html`: Secure login interface with validation and redirection.
- `signup.html`: User registration page with multi-field validation.
- `forgot-password.html`: Password recovery request interface.
- `style.css`: The central design system containing all tokens and styles.

## 📖 How to Run

1. Clone or download the project files.
2. Open `index.html` in any modern web browser (Chrome, Firefox, Edge, Safari).
3. Navigate through the pages using the buttons and links provided.

## 📝 Usage Notes

- **Login Simulation**: Entering a valid email and any password will trigger a success alert and redirect to the home page.
- **Signup Simulation**: Completing the form correctly will trigger a success alert and redirect to the login page.
- **Validation**: If any field is invalid, an error message will appear below the input field and the border will turn red.
